# Product

## Register

brand

## Platform

web

## Users

Recruiters and potential collaborators evaluating Lance's work. They land on the portfolio to judge capability, scan projects for relevance, and decide whether to reach out.

## Product Purpose

A personal portfolio that proves Lance ships real, working software across stacks — from Rust desktop tools to Next.js web apps to Kotlin game mods. Success is a conversation starter: a recruiter or collaborator who sees the work and gets in touch.

## Positioning

AI-accelerated engineer. Lance uses AI as a force multiplier, not a crutch — pairing deep technical understanding with tooling that amplifies output and shortens the build loop.

## Conversion & proof

- Primary CTA: See the work — project cards with live demos and GitHub links.
- Secondary CTA: Contact — email, GitHub, Discord, and a direct form.
- The line a visitor remembers after 10 seconds: "Break it. Fix it. Ship it."
- Belief ladder: (1) Lance knows his stack — proved by the skills inventory and project breadth. (2) He builds things that work — proved by live demos, GitHub repos, and real project descriptions. (3) He's worth contacting — proved by accessible channels and a frictionless form.
- Proof on hand: 3 GitHub repos (Repeatable, StewardshipHub, JellyAddons), 1 live deployment (StewardshipHub), and a Vercel-hosted portfolio itself.

## Brand Personality

Playful and expressive. The portfolio carries personality — bold typography, saturated color fields, floating shapes, and blunt copy. It reads like a poster, not a resume. The voice is direct ("Break it. Fix it. Ship it.") but warm, not aggressive.

## Anti-references

- Generic minimalism: flat beige-and-gray, timid palettes, safe layouts that look like every other developer portfolio.
- SaaS startup templates: gradient-heavy, glassmorphism, soft shadows, interchangeable hero sections.

## Design Principles

- Show, don't tell. Projects are the proof; decoration serves them, not the other way around.
- Personality over polish. Rough edges earn attention. Safe designs are invisible.
- Tools reveal thinking. The stack isn't a buzzword list — each project shows what the tool was used for.
- Clear path to contact. Every page offers a next step; no dead ends.

## Accessibility & Inclusion

Standard best practices: alt text on images, visible focus rings, touch targets ≥44px, reduced-motion fallbacks. No WCAG level targeted.
